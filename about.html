About - Mean Human Vis

This project was created for the course DH2321 Information Visualization VT26, given at KTH Royal Institute of Technology. The project explores interactive visualization of country-level health-, bio-, and socioeconomic attributes. The project was created to support users who want to compare countries or regions, and explore relationships between variables through complementary tools of visualization. The dashboard supports several core analytical tasks like:

  • comparing countries across multiple attributes
  • identifying outliers and patterns
  • exploring relationships between variables and geographical regions
  • filtering and selecting subsets of data

The data

The dataset used in the dashboard was compiled from multiple publicly available sources (see table). These were selected to generate a final dataset that enables meaningful comparisons and that offers data across multiple categories.

The datasets collected contain different data from different years, though they’re not separated by a huge number. The table below shows how we handled the data before converting them to one final, common sheet in Excel. For all datasets, a year was chosen and at least one attribute (column) was extracted together with its corresponding country name and ISO code, all other data not used was left behind (including geographical region means). Using Excel, these separate sheets were then merged by using XLOOKUP, matching the country codes.

The missing data

As the datasets are of different size, not all countries appear across all datasets. Missing data is assigned as null or NaN in the plotly drawing functions. If a missing attribute is chosen it appears as greyed out countries (choropleth map), not at all (scatter plot, bubble chart), or as lines dipping outside of the frame in the parallel coordinates plot (PCP). As there was no way to change the nature of the lines in plotly’s PCP to present missing data, the lines appear to go outside of the PCP frame. It does not adjust the axes for these attributes so the missing data are in these cases distinguishable. In other cases were the axes show values around 0, the lines will still appear visible, however, will still act as a missing value when using the brush tool.
